# Contribuir al proyecto HyPEA-2026

Gracias por querer colaborar. Aquí las reglas básicas para contribuir de forma ordenada.

1. Flujo de trabajo
- Crea una rama para cada cambio: `feature/xxx`, `fix/yyy`, `docs/zzz`.
- Mantén commits pequeños y con mensajes descriptivos.
- Abre un Pull Request (PR) desde tu rama hacia `main` (o la rama principal).

2. Cómo trabajar localmente
- Clona: `git clone https://github.com/gmaxx20/HyPEA-2026.git`
- Crea rama: `git checkout -b feature/nombre`
- Sirve local (HTML estático): `python -m http.server 8000` y abrir `http://localhost:8000`
- Para Jekyll: `bundle exec jekyll serve` (si aplica)

3. PRs y revisiones
- Incluye una descripción clara del cambio.
- Añade pasos para probar los cambios.
- Marca checklist de pruebas realizadas en la descripción del PR.
- Pide al menos 1 revisor antes de mergear.

4. Issues
- Antes de abrir un issue, busca si ya existe.
- Usa plantillas para reportar bugs o proponer mejoras.

5. Estilo y accesibilidad
- Usa HTML semántico.
- Asegúrate de que el sitio sea responsive.
- Añade texto alternativo en las imágenes (`alt`).
- Evita bloques de color con bajo contraste en texto.

6. Licencia y código de conducta
- Respeta la licencia del repositorio.
- Mantén un comportamiento respetuoso y profesional en comentarios y PRs.

Gracias por contribuir — tu ayuda mejora la calidad del proyecto.
